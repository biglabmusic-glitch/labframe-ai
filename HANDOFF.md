# Handoff: LabFrame AI — Telegram Mini App

## 1. Что это

LabFrame AI — Telegram Mini App для русскоязычных зубных техников и керамистов. Пользователь загружает фото зуботехнической работы (коронка / винир / мост) → выбирает стиль обработки → получает готовый визуал для Instagram с правильным фоном, светом, кадрированием, логотипом и текстом к посту.

Полное описание продукта, флоу и ограничений MVP — в папке `specs/`:
- `LabFrame_AI_MVP.md` — общая концепция и принципы продукта
- `LabFrame_AI_TZ_MVP_Telegram_Bot.md` — детальное ТЗ на бот, разделы, БД, AI-логика
- `LabFrame_AI_User_Journey.md` — пользовательский сценарий по шагам
- `color_reference.png` — референс палитры

**Важно:** название `LabFrame AI` и логотип-ромб — рабочие, будут заменены на финальный бренд после утверждения. В коде вынеси их в константы (`APP_NAME`, `BRAND_MARK`), чтобы поменять одним местом.

## 2. О файлах в `design/`

Файлы в этой папке — **дизайн-референсы**, собранные в HTML+React (через Babel inline). Это **не production-код**, а наглядные прототипы того, как должен выглядеть и вести себя интерфейс.

Задача — **воссоздать эти экраны в выбранном стеке** (React + Telegram Web Apps SDK, или Vue, или другой фреймворк, который команда выберет), используя его паттерны и компонентную систему. Если стек ещё не выбран — рекомендуется **React + Vite + TypeScript + Telegram Web Apps SDK (`@twa-dev/sdk`)**, как самый ходовой и хорошо документированный стек для мини-аппов.

Открыть дизайн локально: открой `design/LabFrame AI Mini App.html` в браузере. Все 13 экранов разложены на pan/zoom-холсте.

## 3. Fidelity

**Высокая (hi-fi).** Цвета, типографика, скругления, плотность и размеры таргетов — финальные. Воспроизводи пиксель-в-пиксель в рамках выбранного стека. Иконки и иллюстрации (зуб, превью «до/после») — это упрощённые SVG-плейсхолдеры; в проде их заменят на реальные фотографии работ и финальные иконки.

## 4. Контекст: Telegram Mini App ≠ обычный веб

Дизайн нарисован внутри «фрейма телефона» с собственной шапкой `← LabFrame AI •••` и кнопкой снизу — **это чисто для наглядности мокапа**. В реальном Telegram Mini App:

- **Верхняя шапка** (заголовок + кнопка закрытия / назад + меню `•••`) — рисует сам Telegram, **не приложение**. Цвет/тема настраиваются через `Telegram.WebApp.setHeaderColor()` и `setBackgroundColor()`.
- **Главная кнопка снизу** — это нативная `Telegram.WebApp.MainButton`, не HTML-кнопка. Контролируй через `MainButton.setText()`, `MainButton.setParams({color, text_color})`, `MainButton.onClick()`, `MainButton.show()/hide()`.
- **Кнопка «Назад»** в шапке — `Telegram.WebApp.BackButton`. Подписывайся на `onClick` для управления переходами между экранами.
- **Статус-бар** телефона — нативный системный, не наш.
- **Тема пользователя** — приходит через `Telegram.WebApp.themeParams`. На MVP **фиксируем нашу тёмную палитру**, не подстраиваемся под пользовательскую (иначе ломается визуальная подача).
- **Viewport** — слушай `Telegram.WebApp.onEvent('viewportChanged', ...)`, особенно при появлении клавиатуры (`viewportStableHeight`).
- **Safe areas** — нижний home-indicator iPhone: `padding-bottom: env(safe-area-inset-bottom)`.

Соответственно в production-коде:
- **Не рисовать свою шапку** (она в дизайне есть только для понимания контекста). Контент начинается сразу после нативной шапки Telegram.
- **Не рисовать свою кнопку снизу** — использовать `MainButton`. Цвет = `#93D5E1`, цвет текста = `#0F1221`. Текст и обработчик меняются на каждом шаге флоу.
- **Не рисовать статус-бар и home-indicator** — их рисует Telegram/iOS.

- **Основной шрифт:** `Onest` (Google Fonts, веса 300/400/500/600/700/800)
- **Моноширный:** `JetBrains Mono` (веса 400/500) — для технических подписей, размеров, прогресса, лейблов eyebrow
- **Фолбэк:** `-apple-system, system-ui, sans-serif`

Шкала размеров (px / weight / letter-spacing):
- Хедер экрана H1: `26 / 600 / -0.6`, line-height 1.1
- Заголовок карточки: `15–17 / 600 / -0.2–-0.3`
- Body основной: `13–14 / 400 / -0.1`
- Body вторичный: `11.5–12 / 400 / 0`
- Eyebrow / mono caption: `10–11 / 500–600 / 0.5–0.8`, uppercase, моно

### Иконки

Все иконки — плоские SVG, stroke-only, `strokeWidth=1.6–1.8`, rounded-line/join. Набор примитивов — в `phone.jsx` объект `I`. На проде можно либо повторить эти SVG, либо взять `lucide-react` (стилистически совпадает).

### Размер таргетов

Минимум 44×44px для всего тапаемого. Carousel-карточки в Home — 110px ширина, нормально для скролла.

## 7. Состояние и переходы

### Глобальный state
- `user`: `{ telegramId, username, name, avatar, plan, usage: { used, limit, period } }`
- `brand`: `{ logoUrl?, masterName?, labName?, defaultStyle?, logoPlacement, hashtags: string[] }`
- `currentDraft`: `{ photo?, workType?, style?, branding?, format?, textType?, status }`
- `history`: `Job[]` — список прошлых обработок

### Статусы обработки (из ТЗ §10)
`created → photo_uploaded → settings_selected → processing → done | failed`

### Навигация между шагами
Управляется через `Telegram.WebApp.BackButton`. На каждом экране флоу:
- `BackButton.show()` + `onClick → previous step`
- `MainButton.setText('Далее')` / на последнем шаге — `setText('Сгенерировать пост')` + `showProgress()` при процессинге
- На результате `MainButton.setText('Скачать пост')` и кнопка «Новый пост» как обычный HTML-элемент

### Навигация Home ↔ разделы
Из Home — обычная push-навигация. На внутренних экранах разделов — `BackButton` возвращает на Home. Если мини-апп открыт через deeplink сразу в раздел — `BackButton` закрывает мини-апп (`Telegram.WebApp.close()`).

## 8. Что нужно реализовать на бэке

Из ТЗ §10–11:

### Хранение
- Пользователи: Telegram ID, username, plan, usage counter, дата регистрации
- Бренд-данные: logo blob/URL, имя, лаборатория, хэштеги, предпочтения
- Jobs: исходное фото, настройки, ссылка на результат, статус, timestamps

### AI-пайплайн (2 модуля)
1. **Image AI** — обработка фото (фон, свет, кадр, добавление лого). Жёсткое ограничение из §11.1: **не менять анатомию, форму, цвет, посадку реставрации**. Это критично для доверия пользователей и юридических рисков.
2. **Text AI** — генерация поста: основной текст + 1–2 альтернативы + хэштеги, на русском, по 3 стилям (короткий/продающий/технический).

### Telegram бот
Сам бот (вне мини-аппа) должен принимать `/start`, открывать мини-апп через `web_app` кнопку, и пушить уведомления о готовности обработки (если пользователь свернул).

## 9. Ассеты

В дизайне используются плейсхолдеры:
- **Иконка зуба** — стилизованный SVG-примитив в `phone.jsx` как `I.tooth`. На проде заменить на реальные фотографии работ.
- **`<image-slot>`** — кастомный web-компонент (`image-slot.js`) для drag-and-drop фото в мокап. В реальном приложении не нужен — там обычный `<input type="file">` + рендер preview.
- **Логотип LabFrame AI** — `BrandMark` в `phone.jsx`. Временный, заменить на финальный.
- **Аватары пользователей** — инициалы на цветном фоне (`ИП`). Если у Telegram-пользователя есть фото — использовать его (`Telegram.WebApp.initDataUnsafe.user.photo_url`).

Шрифты: `Onest` и `JetBrains Mono` — Google Fonts, импорт в `tokens.css`.

## 10. Структура файлов в `design/`

- `LabFrame AI Mini App.html` — точка входа, собирает всё через `<DesignCanvas>`
- `tokens.css` — все цвета, скругления, типографика, font-imports
- `phone.jsx` — фрейм телефона + UI-примитивы (`Phone`, `MainButton`, `Pill`, `CircleBtn`, `Card`, `Tag`, `BrandMark`, иконки `I.*`, `StripePlaceholder`)
- `screens-flow.jsx` — 8 экранов основного флоу + хелперы `ScreenIntro`, `StepBadge`
- `screens-aux.jsx` — 5 экранов доп. разделов (Welcome, Home, MyBrand, Examples, Pricing, PhotoHelp)
- `design-canvas.jsx` — pan/zoom canvas для презентации экранов (для прода не нужен)
- `image-slot.js` — placeholder-компонент для фото (для прода не нужен)
